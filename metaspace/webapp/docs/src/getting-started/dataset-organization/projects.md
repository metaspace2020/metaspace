# Projects

## What is a Project?

A project is a way to **organise and share a collection of datasets**,
typically corresponding to a publication or a defined research study.
Any registered user can create a project.

Unlike groups, which are centered on lab membership, projects are centered
on data: a project collects related datasets together, controls who can
access them, and can be linked to a publication once the work is ready to
share publicly.

Projects and groups are independent — a project does not need to belong to
a group, and a group does not need to contain projects. However, they work
well together: a group provides the trusted membership layer, while a
project provides the organisational and sharing layer for datasets.

## Roles

Each project has a **project manager** (the user who created it, or anyone
granted manager rights) and **members**. The manager invites collaborators
and controls access. Members can view and interact with the private datasets
linked to the project.

## Recommended Workflow

It is recommended to create a project before uploading your datasets. This
makes it easier to keep related datasets organised from the start rather
than grouping them retroactively.

1. Create a project (see below)
2. [Upload your dataset](/guides/before-submission/the-upload-page) and link it to the project during submission
3. If you are working in a team, [create or join a group](/getting-started/dataset-organization/groups-and-members) to manage access
   across your datasets and projects

## How to Create a Project

1. Click your username in the top-right corner and select **My projects**
2. Click **Create project**
3. Enter a project name and confirm

Alternatively, projects can be created directly from **My account**.

## How to Invite Members

1. Open your project page
2. Enter the email address of the collaborator you want to add
3. They will receive an invitation to join the project

## Linking a Project to a Publication

Once your work is published, a project can be linked to the corresponding
publication. This makes your datasets discoverable alongside the paper and
is the starting point for making your data publicly available. See
[Publishing Projects](/features/sharing-and-publishing/publishing-projects) for more information.
